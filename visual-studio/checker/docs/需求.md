# 需求文档

## 1. 项目概述
- 项目类型：Visual Studio 2022 扩展程序
- 目标：为 C# 文件提供一键“文件已校验”操作

## 2. 功能需求
### 2.1 右键菜单入口
- 触发条件：当前活动窗口打开的是 C# 文件
- 入口位置：编辑器右键上下文菜单
- 菜单名称：文件已校验

### 2.2 using 排序
- 处理范围：文件头部 using 声明
- 分组规则：按 namespace 的第一个 `.` 之前的部分分组
  1. System
  2. Microsoft
  3. 其他
  4. 当前项目的顶级 namespace
- 组内排序：先按字符串长度升序，长度相同按字典序升序

### 2.3 插入校验模板
- 插入位置：文件末尾
- 模板格式：单行
- 默认模板：`//// 文件已校验，校验时间：@{now:o}`
- 实际示例：`//// 文件已校验，校验时间：@2026-04-16T10:44:42.5159180+08:00`
- 配置要求：模板内容可配置

## 3. 约束与兼容性
- 目标环境：Visual Studio 2022

## 4. 验收标准
- 打开 C# 文件时出现“文件已校验”菜单项，非 C# 文件不出现
- 点击菜单后 using 按指定分组与排序规则重排
- 文件末尾插入一行模板内容，时间格式符合 `now:o`
- 模板可通过配置修改，修改后生效

## 5. 开放问题
- 当前项目的顶级 namespace 取值来源：从当前 C# 文件的 namespace 指令进行硬匹配解析（不使用语法树）
- 模板配置入口位置与格式（例如 VS 选项页或配置文件）
